## 📱 如何测试

### 测试 1：登录和注册

1. 启动开发服务器：
```bash
npm run dev
```

2. 访问 http://localhost:5173

3. 检查右上角是否显示"登录 / 注册"按钮

4. 点击登录，选择：
   - 邮箱密码：输入邮箱和密码注册
   - GitHub：点击"使用 GitHub 继续"

5. 登录成功后应该看到：
   - 用户头像/姓名
   - 下拉菜单
   - "同步数据"、"设置"、"登出"按钮

### 测试 2：回答云端同步

1. 登录后，回答一个问题

2. 点击"保存"按钮

3. 应该看到：
   - 按钮显示"同步中..."
   - 云图标闪烁
   - 保存成功后显示导出对话框

4. 验证数据已上传：
   - 去 Supabase Dashboard
   - Database → answers 表
   - 应该能看到刚才的回答

### 测试 3：成长足迹云端加载

1. 登录后访问"成长足迹"

2. 右上角应该显示：
   - "已同步"（绿色）
   - 或 "同步中..."（蓝色动画）
   - 刷新按钮

3. 点击刷新按钮，应该重新加载数据

4. 应该能看到：
   - 你所有的回答记录
   - 统计数据
   - 时间筛选

### 测试 4：离线模式

1. 回答问题时关闭网络

2. 点击保存

3. 应该仍然能保存到 localStorage

4. 重新联网后，点击用户菜单中的"同步数据"

5. 数据应该上传到云端

## 🔍 调试技巧

### 查看同步状态

打开浏览器控制台（F12），应该能看到这些日志：

```
✅ 上传成功: X 条记录
✅ 下载成功: X 条记录
```

### 检查 Supabase 数据库

Dashboard → Database → 表名 → Table Editor

查看数据是否正确插入。

### 查看网络请求

控制台 → Network 标签
筛选 `supabase` 查看所有 API 请求

## 🐛 可能的问题和解决方案

### 问题 1：登录后没有显示用户信息

**原因**：可能没有正确获取用户 session

**解决**：
```typescript
// 在浏览器控制台运行
localStorage.getItem('sb-zvastmjlcgghgnyqjojt-auth-token')
```

### 问题 2：保存后没有同步到云端

**原因**：
- 未登录
- 网络问题
- RLS 策略限制

**解决**：
1. 确认已登录
2. 检查网络请求
3. 查看 Supabase Dashboard → Auth → Users

### 问题 3：成长足迹页面数据为空

**原因**：
- 还没有回答数据
- 云端没有数据
- 本地也没有数据

**解决**：
1. 先回答几个问题
2. 检查 Supabase Database → answers 表
3. 查看浏览器控制台错误信息

## 📊 数据流程图

```
用户回答问题
    ↓
保存到 localStorage（备份）
    ↓
检查是否登录
    ↓
[是] → 上传到 Supabase
[否] → 仅本地保存
    ↓
显示保存成功
```

```
打开成长足迹页面
    ↓
检查登录状态
    ↓
[已登录] → 从 Supabase 加载数据
[未登录] → 从 localStorage 加载数据
    ↓
显示数据
```
