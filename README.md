## 📖 项目简介

**万万没想到**（Wan Wan Xiang Dao）是一款综合性的个人成长工具，通过精心设计的多元化模块，全方位提升用户的思维能力、创造力和知识水平。

### 🎯 核心理念

- **拓展思维**：打破思维定式，从多个角度思考问题
- **激发创意**：随机组合激发灵感，培养创造力
- **系统学习**：跨学科知识体系，建立完整认知框架
- **持续成长**：记录思考历程，追踪思维变化轨迹
- **健康生活**：科普健康知识，科学管理身心健康

## ✨ 核心功能

### 🤔 思维探索系统

- **双维度分类**：5种思维维度 × 5种生活场景 = 25种思考角度
- **问题库**：数百个精心设计的深度思考问题
- **难度分级**：1-5星难度，渐进式挑战
- **思考记录**：记录每次思考的内容、情绪和感悟
- **智能推荐**：根据历史和偏好推荐问题

**思维维度**：
- `hypothesis` - 假设思维：如果...会怎样？
- `reverse` - 逆向思考：从相反角度思考问题
- `creative` - 联想创意：自由联想激发创意
- `reflection` - 自我反思：反思自身行为和想法
- `future` - 未来设想：畅想未来的可能性

**生活场景**：
- `career` - 职业发展
- `creative` - 创意激发
- `relationship` - 人际关系
- `learning` - 学习成长
- `philosophy` - 生活哲学

### 🎰 灵感老虎机

- **随机词语生成**：三个随机词语组合激发创意
- **彩蛋系统**：特定组合触发特效和奖励
- **创作记录**：保存灵感，支持导出分享
- **快捷操作**：重新抽取、保存、分享等

### 📊 成长追踪

- **时间线视图**：按时间顺序展示思考历程
- **统计面板**：思考次数、类别偏好、活跃趋势
- **思维对比**：对比不同时期对同一问题的思考变化
- **多格式导出**：PDF、Markdown、图片格式

### ⭐ 收藏与待思考

- **收藏系统**：收藏感兴趣的问题，便于深入思考
- **待思考列表**：标记当前不想思考的问题，稍后处理
- **集合管理**：自定义收藏集合，灵活组织问题
- **拖拽排序**：支持拖拽排序，优化浏览体验

### 🧩 逻辑推理游戏

四大益智游戏，全方位锻炼逻辑思维：

1. **海龟汤** 🧩
   - 通过是/否问题推理离奇故事真相
   - 悬疑推理，锻炼逻辑推理能力

2. **谜语人** 💡
   - 猜谜语，锻炼联想思维和文字理解
   - 有提示系统，降低难度

3. **Yes or No** ❓
   - AI出题，通过是/否问题猜出目标词语
   - 多个词库类别：动物、食物、物品、职业等

4. **猜数字** 🔢
   - 根据xAxB提示猜出四位不重复数字
   - 策略建议，帮助优化推理过程

### 💚 知识科普

三大知识模块 + AI智能问答：

1. **世界之最** 🏆
   - 10个世界纪录和自然奇观知识点
   - 涵盖地理、自然、动物、科学等领域

2. **系统思维** 🕸️
   - 12个跨学科知识
   - 涵盖心理学、社会学、科学、数学、哲学、医学
   - 支持按学科分类筛选

3. **健康主理** 💓
   - 12个重要健康知识
   - 涵盖睡眠、运动、饮食、心理健康等
   - 快速跳转功能，方便查找

4. **AI 智能问答** 🤖
   - 用户提问，AI智能解答
   - 支持示例问题快速开始
   - 流畅的聊天界面

### ✍️ 写作创造

两大创作模式，提升写作能力：

1. **灵感老虎机** ✨
   - 随机词语组合激发创作灵感
   - 记录创作内容

2. **文笔挑战** 📝
   - 根据开头句续写精彩故事
   - 锻炼文笔和情节构思能力

### 📅 每日签到

- **日历视图**：可视化展示签到记录
- **连续统计**：追踪连续签到天数
- **激励系统**：签到成就和里程碑
- **绿色主题**：清新自然的视觉设计

### 👤 个人中心

- **数据统计**：全面的使用统计和成就展示
- **数据导出**：导出个人数据备份
- **数据导入**：从备份恢复数据
- **设置管理**：个性化设置选项

## 🚀 快速开始

### 环境要求
- Node.js >= 18.0.0
- npm >= 9.0.0

### 安装依赖
```bash
git clone <repository-url>
cd万万没想到
npm install
```

### 配置环境变量
创建 `.env` 文件：
```bash
VITE_SUPABASE_URL=your_supabase_url
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key
```

### 启动开发服务器
```bash
npm run dev
```

应用将在 `http://localhost:5173` 启动

⚠️ **注意**：开发服务器必须运行在端口 5173

如果端口被占用：
```bash
# Windows
netstat -ano | findstr :5173
taskkill /PID <pid> /F

# Unix/Linux/Mac
lsof -ti:5173 | xargs kill -9
```

### 构建生产版本
```bash
npm run build
```

### 预览生产构建
```bash
npm run preview
```

## 🔧 开发指南

### 路径别名

```typescript
@/              → ./src/
@components/    → ./src/components/
@hooks/         → ./src/hooks/
@utils/         → ./src/utils/
@types/         → ./src/types/
@constants/     → ./src/constants/
@stores/        → ./src/stores/
```

### 代码规范

#### 命名约定
- 组件：PascalCase（如 `QuestionCard.tsx`）
- Hooks：camelCase + use 前缀（如 `useAuth.ts`）
- 常量：PascalCase（如 `MAX_QUESTIONS`）
- 工具函数：camelCase（如 `formatDate`）

#### 文件组织
- 一个文件一个组件
- 组件文件使用 `.tsx` 扩展名
- 工具函数使用 `.ts` 扩展名
- 类型定义集中管理

### 状态管理

- **全局状态**：使用 Zustand store
- **本地状态**：使用 React Hooks
- **持久化**：localStorage + Zustand persist 中间件

### 关键约束

#### 端口管理（必须遵守）
⚠️ **开发服务器必须运行在端口 5173**

如果端口被占用：
1. 终止占用进程
2. 重新在5173端口启动
3. 禁止使用5174等其他端口

#### 代码修改安全
✅ **不得破坏现有功能**

修改前：
1. 识别所有受影响的功能
2. 制定测试计划
3. 准备回滚方案

修改后：
1. 全面测试相关功能
2. 发现问题立即修复
3. 确保全部功能正常

#### 磁盘空间管理
💾 **优先使用 D 盘**

- 临时文件：`D:\Code\data\temp`
- 导出文件：`D:\Code\data\exports`
- 避免占用C盘空间
